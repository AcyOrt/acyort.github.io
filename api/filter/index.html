Filter

    OverviewFilterHelperMethods

A filter is used to modify some specified data. AcyOrt passes data to filters in sequence and the filters then modify the data one after the other.

Here are list of filters used by AcyOrt.

Support list

after_init

Executed after AcyOrt initialize

acyort.extend.register('after_init', function() {
  // ...
})

after_fetch

Executed after getting issues data form GitHub

acyort.extend.register('after_fetch', function(data) {
  // data is array of github issues data
  // ...
})

after_process

Executed after processing the posts data

acyort.extend.register('after_process', function(data) {
/*
{
  posts: [...],
  pages: [...],
  categories: [...],
  tags: [...],
  index: [...],
  category: {
    ...: [...]
  },
  tag: {
    ...: [...]
  }
}
*/
})

after_build

Executed after generating the static files

acyort.extend.register('after_build', function(data) {
/*
{
  posts: [...],
  pages: [...],
  categories: [...],
  tags: [...],
  index: [...],
  category: {
    ...: [...]
  },
  tag: {
    ...: [...]
  }
}
*/
})

Promise use

Filters support Promise

acyort.extend.register('after_init', function() {
  return new Promise((resolve) => {
    // ...
    setTimeout(resolve, 1000)
  })
})
